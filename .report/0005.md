# Инкремент 0005 — B5: минимальный ACPI 1.0

**Дата:** 2026-04-30
**Этап:** B (i386-совместимость с NetBSD), подэтап B5.
**Статус:** Реализация.
**Зависит от:** [0001.md](0001.md), [0002.md](0002.md), [0003.md](0003.md), [0004.md](0004.md).

---

## 1. Цель

NetBSD/i386 11.0 GENERIC при boot ищет ACPI до MP-table. Если найден `RSD PTR ` — идёт по нему в RSDT, парсит MADT/FADT, и предпочитает топологию из ACPI вместо MP-table из 0003. Если RSDP отсутствует — fallback на MP-table.

**Цель:** разместить корректный набор ACPI 1.0 таблиц так, чтобы ОС нашла RSDP, прошла RSDT → MADT → FADT → DSDT, прочитала минимальный AML и продолжила boot. **Реальные ACPI методы не реализованы** — DSDT содержит только пустой `Scope(\_SB) {}`, что валидно с точки зрения парсера, но не предоставляет device discovery, power management или GPE handling.

Минимальный инкремент: код + JUnit-тесты + зелёная сборка. **Без реального запуска NetBSD ISO.**

## 2. Размещение

Решение: **отдельный 4 KB блок на `0xFB000-0xFBFFF`** (соседний к MpTable из 0003 на `0xFC000`).

Все 5 таблиц упакованы в один блок с фиксированными смещениями:

| Внутри блока | Физ. адрес | Размер | Таблица |
|--------------|------------|--------|---------|
| `0x000`      | `0xFB000`  | 20     | RSDP    |
| `0x020`      | `0xFB020`  | 44     | RSDT (header + 2 указателя) |
| `0x050`      | `0xFB050`  | 64     | MADT    |
| `0x090`      | `0xFB090`  | 116    | FADT    |
| `0x108`      | `0xFB108`  | 43     | DSDT    |
| `0x133–0xFFF`| –           | rest   | оригинальные байты Bochs BIOS |

`0xFB000` — внутри обязательного диапазона сканирования RSDP (0xE0000–0xFFFFF), 16-байт выровнен (и даже 4 KB-выровнен для `mapMemory`).

## 3. Содержимое таблиц

### 3.1. RSDP (20 байт, ACPI 1.0)

| Off | Поле           | Значение                   |
|-----|----------------|-----------------------------|
| 0   | Signature      | `RSD PTR ` (с пробелом, 8 ASCII) |
| 8   | Checksum       | вычисляется (∑ по 20 байт ≡ 0) |
| 9   | OEM ID         | `JPCJPC` (6 ASCII)          |
| 15  | Revision       | `0x00` (ACPI 1.0)           |
| 16  | RSDT addr      | `0xFB020` (LE32)            |

### 3.2. SDT header (36 байт) — общий формат RSDT/MADT/FADT/DSDT

| Off | Поле               | Размер |
|-----|---------------------|--------|
| 0   | Signature           | 4      |
| 4   | Length              | 4 LE32 |
| 8   | Revision            | 1      |
| 9   | Checksum            | 1      |
| 10  | OEM ID              | 6      |
| 16  | OEM Table ID        | 8      |
| 24  | OEM Revision        | 4 LE32 |
| 28  | Creator ID          | 4      |
| 32  | Creator Revision    | 4 LE32 |

Для всех таблиц: OEM ID = `JPCJPC`, OEM Table ID = `JPCi386 `, OEM Rev = 1, Creator ID = `JPC `, Creator Rev = 1.

### 3.3. RSDT — `RSDT`

После header — массив 32-битных указателей на остальные SDT.

| Entry | Указывает на     |
|-------|-------------------|
| 0     | MADT @ 0xFB050    |
| 1     | FADT @ 0xFB090    |

Length = 36 + 4·2 = **44 байт**. Revision = 1.

### 3.4. MADT — `APIC` (Multiple APIC Description Table)

| Off (от начала MADT) | Поле | Значение |
|----------------------|-------|----------|
| 0    | SDT header           | sig=`APIC`, len=64, rev=1 |
| 36   | Local APIC address   | `0xFEE00000` (LE32)       |
| 40   | Flags                | `0x00000001` (PCAT_COMPAT — 8259 присутствует) |
| 44   | Processor LAPIC entry | type=0, len=8, ACPI proc ID=0, APIC ID=0, flags=1 (enabled) |
| 52   | IO APIC entry        | type=1, len=12, IO APIC ID=2, reserved=0, addr=`0xFEC00000`, GSI base=0 |

Length = 36 + 8 + 8 + 12 = **64 байта**. Согласовано с MP-table из 0003.

### 3.5. FADT — `FACP` (Fixed ACPI Description Table, ACPI 1.0)

| Off  | Поле                  | Значение |
|------|------------------------|----------|
| 0    | SDT header             | sig=`FACP`, len=116, rev=1 |
| 36   | FIRMWARE_CTRL          | 0 (нет FACS — отдельной зоны для wake vectors) |
| 40   | DSDT                   | `0xFB108` |
| 44   | INT_MODEL              | 0 (dual 8259 PIC) |
| 45   | reserved               | 0 |
| 46   | SCI_INT                | 9 (LE16) |
| 48   | SMI_CMD                | 0 (нет SMI) |
| 52   | ACPI_ENABLE            | 0 |
| 53   | ACPI_DISABLE           | 0 |
| 54   | S4BIOS_REQ             | 0 |
| 55   | reserved               | 0 |
| 56   | PM1a_EVT_BLK           | 0 (PM не реализован) |
| 60   | PM1b_EVT_BLK           | 0 |
| 64   | PM1a_CNT_BLK           | 0 |
| 68   | PM1b_CNT_BLK           | 0 |
| 72   | PM2_CNT_BLK            | 0 |
| 76   | PM_TMR_BLK             | 0 |
| 80   | GPE0_BLK               | 0 |
| 84   | GPE1_BLK               | 0 |
| 88   | PM1_EVT_LEN            | 0 |
| 89   | PM1_CNT_LEN            | 0 |
| 90   | PM2_CNT_LEN            | 0 |
| 91   | PM_TM_LEN              | 0 |
| 92   | GPE0_BLK_LEN           | 0 |
| 93   | GPE1_BLK_LEN           | 0 |
| 94   | GPE1_BASE              | 0 |
| 95   | reserved               | 0 |
| 96   | P_LVL2_LAT             | 0 (LE16) |
| 98   | P_LVL3_LAT             | 0 (LE16) |
| 100  | FLUSH_SIZE             | 0 (LE16) |
| 102  | FLUSH_STRIDE           | 0 (LE16) |
| 104  | DUTY_OFFSET            | 0 |
| 105  | DUTY_WIDTH             | 0 |
| 106  | DAY_ALRM               | 0 |
| 107  | MON_ALRM               | 0 |
| 108  | CENTURY                | 0 |
| 109  | reserved (3)           | 0 |
| 112  | Flags                  | 0 (LE32) |

Length = **116 байт**. Revision = 1.

### 3.6. DSDT — `DSDT` + AML

| Off | Содержание |
|-----|------------|
| 0   | SDT header (36 байт), sig=`DSDT`, rev=1 |
| 36  | AML body: 7 байт |

AML body:
```
0x10              ScopeOp
0x06              PkgLength = 6 (всего: PkgLength + NameString + TermList)
0x5C              RootChar '\'
0x5F 0x53 0x42 0x5F   NameSeg "_SB_"
                  TermList = пусто
```

Это валидный минимальный AML: `Scope (\_SB) {}` — пустой системный «корень шины». Парсер пройдёт его без ошибок.

Length = **43 байта**.

### 3.7. Контрольные суммы

Все SDT и RSDP имеют checksum-поля. Вычисление:
- Установить checksum в 0.
- Просуммировать все байты таблицы (в RSDP — все 20).
- Checksum = `(byte)(-sum)` ⇒ итоговая сумма ≡ 0 mod 256.

## 6. Что НЕ входит

- **FACS** (Firmware ACPI Control Structure) — wake-vector для S3/S4. Не нужен для GENERIC boot.
- **SSDT** — дополнительные AML-таблицы. Минимальный DSDT покрывает.
- **AML методы:** `_PIC` (выбор APIC vs PIC), `_PRT` (PCI IRQ routing), `_S5` (shutdown). Можно добавить инкрементом.
- **PM register block** — PM1a_EVT/CNT, PM_TMR. PM_TMR_BLK=0 ⇒ ОС не пользуется ACPI таймером.
- **GPE** (General Purpose Events).
- **Реальный boot NetBSD** (B7).
- **Согласование DSDT с реальной топологией PCI** — гость не получит правильного IRQ routing для PCI устройств от ACPI; будет fallback на MP-table или legacy PIC.

## 7. Риски

- **Overlay рушит первые 0x133 байта оригинального BIOS на 0xFB000.** Аналогично 0003 — захватываем оригинал, накладываем поверх первых ~310 байт; остаток 4 KB сохраняется. Если Bochs BIOS использует эту область — поломка, но 0xFB000 ≠ reset vector и ≠ традиционные BIOS-данные.
- **Дублирование топологии MADT vs MP-table.** Если ОС читает обе и они расходятся — undefined behavior. Мы делаем их согласованными (LAPIC@0xFEE00000, IO-APIC@0xFEC00000, ID=2, BSP=0).
- **NetBSD может предпочесть ACPI и закопаться в DSDT, не найдя устройств.** Минимальный пустой `\_SB` валиден синтаксически; некоторые версии NetBSD ругаются «empty namespace», но не падают.
- **Контрольные суммы.** Если хотя бы в одной таблице чек-сумма неверна, ОС отвергает таблицу как corrupt. Тесты проверяют все 5 чек-сумм.

## 9. Предложения по улучшению

- **Унификация overlay-механизма.** MpTable, Acpi и (возможно, в будущем) другие таблицы делают одно и то же: захват оригинала + overlay + EPROM mapping. Стоит выделить общий `BiosShadowOverlay` базовый класс.
- **Вынести checksum/SDT-builder утилиты.** В обоих 0003 (MP) и 0005 (ACPI) есть LE16/LE32 writers и checksum-finalizer; кандидат на `org.jpc.emulator.motherboard.TableUtils`.
- **`_PIC` метод в DSDT** — позволил бы ОС переключаться между PIC и APIC. Полезно после доделки B3 до working LAPIC. ~10 байт AML.
- **`_S5` package** — Shutdown через ACPI. Полезно для CI-driven boot/halt тестов.
- **PM_TMR_BLK** — IO-порт ACPI таймера (типично 0x408). Если эмулировать — Linux/NetBSD используют как stable time source. Связано с возможной доработкой `IOPortHandler`.
- **Согласовать MP-table и ACPI MADT.** Сейчас они построены независимо с одними и теми же константами. Лучше — single source of truth (например, в `Acpi` или `MpTable` ссылаются на общую `PlatformTopology`).
